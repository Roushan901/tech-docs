AWS (Amazon Web Services)

Complete guide to AWS documentation, services, and best practices for cloud computing.

Overview‚Äã

Amazon Web Services (AWS) is the world's most comprehensive and broadly adopted cloud platform, offering over 200 fully-featured services from data centers globally.

Getting Started with AWS‚Äã

Prerequisites‚Äã

  • AWS Account (sign up at aws.amazon.com)
  • Basic understanding of cloud computing
  • AWS CLI installed (optional but recommended)

AWS Account Setup‚Äã

1. Create AWS Account‚Äã

  1. Visit aws.amazon.com
  2. Click "Create an AWS Account"
  3. Provide email and password
  4. Add payment information
  5. Verify identity

2. Set Up MFA (Multi-Factor Authentication)‚Äã

# Install AWS CLI
pip install awscli

# Configure AWS credentials
aws configure

3. Create IAM User‚Äã

  • Navigate to IAM Console
  • Create new user
  • Attach policies
  • Download access keys

Core AWS Services‚Äã

Compute Services‚Äã

EC2 (Elastic Compute Cloud)‚Äã

Virtual servers in the cloud for running applications.

Key Features:

  • Scalable computing capacity
  • Multiple instance types
  • Pay-as-you-go pricing
  • Auto Scaling

Launch EC2 Instance

# Using AWS CLI
aws ec2 run-instances \
  --image-id ami-0c55b159cbfafe1f0 \
  --instance-type t2.micro \
  --key-name MyKeyPair \
  --security-group-ids sg-xxxxxxxx \
  --subnet-id subnet-xxxxxxxx

Instance Types:

TypeUse CasevCPUMemory
t2.microDevelopment11 GB
t2.smallSmall apps12 GB
m5.largeGeneral purpose28 GB
c5.xlargeCompute optimized48 GB

Lambda‚Äã

Serverless compute service for running code without managing servers.

Create Lambda Function

# lambda_function.py
def lambda_handler(event, context):
    return {
        'statusCode': 200,
        'body': 'Hello from Lambda!'
    }

Deploy with AWS CLI

# Create deployment package
zip function.zip lambda_function.py

# Create Lambda function
aws lambda create-function \
  --function-name my-function \
  --runtime python3.9 \
  --role arn:aws:iam::account-id:role/lambda-role \
  --handler lambda_function.lambda_handler \
  --zip-file fileb://function.zip

Storage Services‚Äã

S3 (Simple Storage Service)‚Äã

Object storage service for storing and retrieving any amount of data.

Create S3 Bucket

# Create bucket
aws s3 mb s3://my-bucket-name

# Upload file
aws s3 cp file.txt s3://my-bucket-name/

# Download file
aws s3 cp s3://my-bucket-name/file.txt ./

# List bucket contents
aws s3 ls s3://my-bucket-name/

S3 Bucket Policy

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "PublicReadGetObject",
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::my-bucket-name/*"
    }
  ]
}

EBS (Elastic Block Store)‚Äã

Block storage volumes for EC2 instances.

Create EBS Volume

aws ec2 create-volume \
  --availability-zone us-east-1a \
  --size 100 \
  --volume-type gp3

Database Services‚Äã

RDS (Relational Database Service)‚Äã

Managed relational database service.

Supported Databases:

  • MySQL
  • PostgreSQL
  • Oracle
  • SQL Server
  • MariaDB
  • Amazon Aurora

Create RDS Instance

aws rds create-db-instance \
  --db-instance-identifier mydbinstance \
  --db-instance-class db.t3.micro \
  --engine mysql \
  --master-username admin \
  --master-user-password mypassword \
  --allocated-storage 20

DynamoDB‚Äã

NoSQL database service for single-digit millisecond performance.

Create Table

aws dynamodb create-table \
  --table-name Users \
  --attribute-definitions \
    AttributeName=UserId,AttributeType=S \
  --key-schema \
    AttributeName=UserId,KeyType=HASH \
  --billing-mode PAY_PER_REQUEST

Put Item

aws dynamodb put-item \
  --table-name Users \
  --item '{
    "UserId": {"S": "user123"},
    "Name": {"S": "John Doe"},
    "Email": {"S": "john@example.com"}
  }'

  --tag-specifications 'ResourceType=vpc,Tags=[{Key=Name,Value=MyVPC}]'

Security Services‚Äã

IAM (Identity and Access Management)‚Äã

Manage access to AWS services and resources.

IAM Policy Example

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "s3:GetObject",
        "s3:PutObject"
      ],
      "Resource": "arn:aws:s3:::my-bucket/*"
    }
  ]
}

Create IAM User

# Create user
aws iam create-user --user-name developer

# Attach policy
aws iam attach-user-policy \
  --user-name developer \
  --policy-arn arn:aws:iam::aws:policy/ReadOnlyAccess

Pipeline Structure:

Source (GitHub) ‚Üí Build (CodeBuild) ‚Üí Test ‚Üí Deploy (ECS)

CloudFormation‚Äã

Infrastructure as Code (IaC) service.

Template Example

AWSTemplateFormatVersion: '2010-09-09'
Resources:
  MyBucket:
    Type: AWS::S3::Bucket
    Properties:
      BucketName: my-unique-bucket-name
      PublicAccessBlockConfiguration:
        BlockPublicAcls: true

Deploy Stack

aws cloudformation create-stack \
  --stack-name my-stack \
  --template-body file://template.yaml

AWS Best Practices‚Äã

Cost Optimization‚Äã

  • Use Reserved Instances for predictable workloads
  • Implement Auto Scaling
  • Delete unused resources
  • Use S3 lifecycle policies
  • Monitor with AWS Cost Explorer

Security Best Practices‚Äã

  • Enable MFA for all users
  • Use IAM roles instead of access keys
  • Encrypt data at rest and in transit
  • Regular security audits
  • Implement least privilege access

Reliability‚Äã

  • Deploy across multiple Availability Zones
  • Implement backup and disaster recovery
  • Use Auto Scaling and Load Balancing
  • Monitor with CloudWatch
  • Regular testing of failover procedures

Performance‚Äã

  • Use CloudFront for content delivery
  • Implement caching strategies
  • Choose appropriate instance types
  • Optimize database queries
  • Monitor performance metrics

Monitoring & Logging‚Äã

CloudWatch‚Äã

Monitor AWS resources and applications.

Create Alarm

aws cloudwatch put-metric-alarm \
  --alarm-name high-cpu \
  --alarm-description "Alert when CPU exceeds 80%" \
  --metric-name CPUUtilization \
  --namespace AWS/EC2 \
  --statistic Average \
  --period 300 \
  --threshold 80 \
  --comparison-operator GreaterThanThreshold

AWS Pricing Models‚Äã

  • Savings Plans - Flexible pricing model
